# MVP Agent Option Audit

This is the gate between "all rows are at least MVP Working" and the parity/production pass. Before any feature is promoted to `Parity Candidate` or `Production Ready`, ProxyForge must prove that every operator workflow has an agent-facing command, a documented schema, or an explicit tracked gap.

## Audit Rule

- Every GUI/runtime option that can gather, view, replay, scan, exploit, automate, export, or report must be reachable through `scripts/proxyforge-agent.mjs` or explicitly listed as a deferred parity gap.
- Operational commands preserve full tokens, keys, cookies, headers, callbacks, raw requests, raw responses, transcripts, and backend manifests. Redaction is only for reporting/submission commands.
- Active commands require scope and either `--execute` or an approval artifact.
- The audit is repeated after every remaining `Partial` row reaches `MVP Working`.

## Explicit Post-MVP Check

Run this check immediately after the last `Partial` feature-matrix row reaches `MVP Working`:

```bash
npm run test:agent
npm run test:release
node tests/agent-option-audit.mjs
```

The check must confirm:

- The CLI exposes every command in the coverage table.
- Codex, Claude, Vantix, schema, and top-level agent docs mention Intruder and Repeater desync/race commands.
- `docs/AGENTIC_INTERFACE.md` still states that executor outputs preserve full secrets and raw traffic until the reporting phase.
- The feature matrix keeps the agent audit as a mandatory gate before parity/production.
