ProxyForge Release Checklist

This checklist tracks the desktop proof needed before a Linux or Windows build can be called Production Ready. A package is not release-ready just because electron-builder produced a file; each lane needs build, install, runtime, and security smoke evidence.

Alpha Publication Gates

Source alpha status is limited to 0.1.0-alpha.1 source archives and GitHub tags. Create or publish the source alpha tag only after the GitHub source gate installs Chromium with npx playwright install --with-deps chromium and records npm run test:ci:fast passing for the exact commit. If a local extracted archive cannot download Playwright Chromium, record that as environment-blocked; it does not replace the required GitHub source-gate receipt.

Binary builds are allowed before certification. Treat AppImage, deb, NSIS, and portable outputs as unsigned validation artifacts until the exact release tag has native Linux and Windows artifacts, packaged Electron fuse receipts, packaged-license receipts, native release-smoke receipts, live-validation evidence, release/SHA256SUMS.txt, and a passing npm run release:preflight. Do not label binaries as public release-certified installers before this preflight passes.

npm run test:release compiles the renderer and Electron runtime, then verifies the release scripts, packaged entrypoints, builder targets, ignored artifact directories, and this checklist.

npm run release:fuses verifies the source-side Electron fuse policy is wired into Electron Builder through scripts/electron-fuse-policy.mjs. npm run test:release:fuses verifies the policy itself: NODE_OPTIONS, inspector CLI arguments, and extra file:// privileges are disabled; cookie encryption, embedded app.asar integrity validation, and app.asar-only loading are enabled; and the temporary RunAsNode compatibility exception is explicit. Native release jobs must additionally run node scripts/electron-fuse-policy.mjs --verify --app <packaged Electron executable> against the exact Linux or Windows artifact.

npm run release:preflight is the final public-publish guard. Run it only after live validation and the native Linux/Windows artifact matrix have completed for the exact release tag. It requires test-results/live-validation-summary.json with releaseDecision: GO, present live-validation artifact paths under test-results/, Windows live-validation evidence and log output, the release tag peeled to the checked-out commit, Linux and Windows Electron fuse receipts, packaged-license receipts, release-smoke receipts, all four distributable artifacts, and release/SHA256SUMS.txt entries whose SHA-256 values match the actual artifacts.

npm run release:live:source runs the local authorized live-validation workflow through the real proxyforge-agent interface before public publish. It starts a loopback HTTP/HTTPS fixture, drives ProxyForge MITM capture through a real proxy listener, imports captured evidence, executes original and mutated replay traffic, runs scoped active scanner checks, validates local HTTP/DNS/SMTP OAST callbacks, exports JSON/Markdown reports, verifies report redaction of pf_live_validation_token and pf_live_validation_secret, and writes test-results/live-validation-summary.json plus the test-results/live-* evidence directories. npm run release:live:windows runs the same headless live workflow on a Windows node and writes test-results/windows-live-validation/; GUI/RDP smoke still needs to be recorded separately when required for final publish evidence.
